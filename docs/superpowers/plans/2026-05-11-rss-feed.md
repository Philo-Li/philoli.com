# RSS Feed Support Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add three RSS feeds (en, zh, zh-TW) with full-content posts, footer + blog-index entry points, and per-page auto-discovery.

**Architecture:** A small `src/lib/rss.ts` module owns markdown→HTML conversion, URL absolutization, and per-locale post filtering. Three thin endpoints (`src/pages/rss.xml.ts`, `src/pages/zh/rss.xml.ts`, `src/pages/zh-TW/rss.xml.ts`) call `@astrojs/rss` with items produced by that module. UI changes are localized: a `<link rel="alternate">` in `SEOHead`, an RSS link in `Footer`, an "RSS" button in `BlogIndexPage`.

**Tech Stack:** Astro 6 static output, content collections, `@astrojs/rss`, `markdown-it`, `sanitize-html`, vitest.

**Spec:** [docs/superpowers/specs/2026-05-11-rss-feed-design.md](../specs/2026-05-11-rss-feed-design.md)

- [ ] **Step 3: Implement `renderMarkdownForRss` and `extractPlainDescription`**

Create `src/lib/rss.ts`:

```ts
import MarkdownIt from 'markdown-it';
import sanitizeHtml from 'sanitize-html';

const md = new MarkdownIt({ html: true, linkify: true, typographer: false });

/**
 * Convert post markdown to HTML suitable for <content:encoded>.
 * Root-relative URLs are rewritten to absolute (so RSS readers can load images).
 */
export function renderMarkdownForRss(body: string, siteUrl: string): string {
  // Strip <!--more--> so it doesn't leak into output as a literal comment.
  const cleaned = body.replace(/<!--\s*more\s*-->/g, '');
  const rendered = md.render(cleaned);
  const absolutized = absolutizeUrls(rendered, siteUrl);
  return sanitizeHtml(absolutized, {
    allowedTags: sanitizeHtml.defaults.allowedTags.concat([
      'img', 'figure', 'figcaption', 'h1', 'h2',
    ]),
    allowedAttributes: {
      ...sanitizeHtml.defaults.allowedAttributes,
      img: ['src', 'alt', 'title', 'width', 'height'],
      a: ['href', 'name', 'target', 'rel', 'title'],
    },
    allowedSchemes: ['http', 'https', 'mailto'],
  });
}

function absolutizeUrls(html: string, siteUrl: string): string {
  const origin = siteUrl.replace(/\/$/, '');
  return html
    .replace(/\s(src|href)="\/(?!\/)/g, ` $1="${origin}/`);
}

/**
 * Pick a short plain-text summary for <description>.
 * Order: frontmatter `description` → text before <!--more--> → first ~200 chars.
 */
export function extractPlainDescription(
  body: string,
  frontmatterDescription: string | undefined,
): string {
  if (frontmatterDescription && frontmatterDescription.trim().length > 0) {
    return frontmatterDescription.trim();
  }
  const moreIdx = body.search(/<!--\s*more\s*-->/);
  const slice = moreIdx >= 0 ? body.slice(0, moreIdx) : body;
  const plain = stripMarkdown(slice).trim();
  if (moreIdx >= 0) return plain;
  if (plain.length <= 200) return plain;
  return plain.slice(0, 200).trimEnd() + '…';
}

function stripMarkdown(input: string): string {
  return input
    .replace(/```[\s\S]*?```/g, '')          // fenced code
    .replace(/`[^`]*`/g, '')                 // inline code
    .replace(/!\[[^\]]*]\([^)]*\)/g, '')     // images
    .replace(/\[([^\]]*)]\([^)]*\)/g, '$1')  // links → text
    .replace(/^#{1,6}\s+/gm, '')             // headings
    .replace(/[*_]{1,3}([^*_]+)[*_]{1,3}/g, '$1') // bold/italic
    .replace(/^>\s?/gm, '')                  // blockquote markers
    .replace(/<!--[\s\S]*?-->/g, '')         // HTML comments
    .replace(/\r?\n+/g, ' ')                 // collapse newlines
    .replace(/\s+/g, ' ');
}
```

    description: '關於藝術、生活、投資和創造的思考。',
    language: 'zh-TW',
  },
};

export function feedLocaleFor(locale: string | undefined): FeedLocale {
  if (locale === 'zh') return 'zh';
  if (locale === 'zh-TW') return 'zh-TW';
  return 'en';

  return fl === 'en' ? '/rss.xml' : `/${fl}/rss.xml`;
}

## Task 4: Helper module — `buildFeedItems(feedLocale, site)`

**Files:**
- Modify: `src/lib/rss.ts` (append)

This is the bridge between Astro's content collection and `@astrojs/rss`. It loads `getCollection('blog')`, filters to entries whose `id` starts with `<feedLocale>/`, sorts by `date` descending, and returns an array of items shaped for `@astrojs/rss`. We won't unit-test this directly (it depends on `astro:content`); we verify it through the build.

- [ ] **Step 1: Implement `buildFeedItems`**

Append to `src/lib/rss.ts`:

```ts
import { getCollection } from 'astro:content';
import type { RSSFeedItem } from '@astrojs/rss';
import { DEFAULT_LOCALE, localizedPath } from '../i18n';

const POST_LOCALE_FOR_FEED: Record<FeedLocale, string> = {
  en: DEFAULT_LOCALE,  // 'en'
  zh: 'zh',
  'zh-TW': 'zh-TW',
};

export async function buildFeedItems(
  feedLocale: FeedLocale,
  site: URL | string,
): Promise<RSSFeedItem[]> {
  const siteUrl = typeof site === 'string' ? site : site.toString();
  const origin = siteUrl.replace(/\/$/, '');
  const prefix = `${feedLocale}/`;
  const routeLocale = POST_LOCALE_FOR_FEED[feedLocale];

  const all = await getCollection('blog');
  const posts = all
    .filter(p => p.id.startsWith(prefix))
    .sort((a, b) => b.data.date.valueOf() - a.data.date.valueOf());

  return posts.map(post => {
    const slug = post.id.slice(prefix.length);
    const link = localizedPath(`/blog/${slug}`, routeLocale);
    const tags = Array.isArray(post.data.tags)
      ? post.data.tags
      : post.data.tags ? [post.data.tags] : [];
    return {
      title: post.data.title,
      pubDate: post.data.date,
      link,
      description: extractPlainDescription(post.body ?? '', post.data.description),
      content: renderMarkdownForRss(post.body ?? '', origin),
      categories: tags,
    };
  });
}
```

export async function GET(context: APIContext) {
